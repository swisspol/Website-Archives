Displaying WaterRace FAQ

> I have a Voodoo 4000 or 5000 with the 1.1.2 drivers, but you say I need the 1.1.3 which are not out yet.

Actually the Rave driver provided with the 1.1.2 driver set, is the 1.1.1, and it still has some bugs which can cause the game to crash. Everything should be fixed in the 1.1.3, which should be out in a few days.

> I do have a Voodoo 3/4/5 that I use as a second screen and I want to play WaterRace on it, but I don't see the characters in the interface.

There is a bug in the 3Dfx Rave driver which prevents rendering contexts on secondary screens to appear correctly: set your Voodoo 3/4/5 screen as the main screen (the one with the menubar).

> Altough I own the original WaterRace CD and it appears mounted on the desktop, when I launch the game, I get an error that says the WaterRace CD is not inserted, and the game is locked in multiplayer mode.

Make sure you put the CD inside a standard Mac CD-Rom drive that uses the standard Apple CD Driver. Make sure you have the latest update of WaterRace which fixes some bugs regarding CD detection. As a last resort, disable any third party CD Driver.

> I find the controls / gameplay to be difficult in this game.

The physic engine in WaterRace is quite realistic for an arcade game and this might disturb some people. We strongly recommend you to read the "Ship driving tips" text file that comes with the demo: you will learn the few tips necessary to control the ships perfectly!

The most important tip is: always keep the engine power to the max (you will have much better control over the ship), don't be affraid of speed!

> I can't play on some levels like Artic or New York: there's to much fog; the screen is completely white or black.

You need to install OpenGL 1.2.1 from Apple. It contains ATI drivers that fix a bug regarding the fog [ OpenGL 1.2.1 English | OpenGL 1.2.1 French ].

> When I launch WaterRace, it says I need to install QuickDraw 3D 1.6 even if it is already installed.

Remove the "Classic Rave" extension from the Extensions folder of the system folder. It is installed by MacOS X Beta and conflicts with the normal "QuickDraw 3D Rave" extension.

> I do have a Voodoo 3 with the b12 drivers installed and when I launch the game I get a "Not enough VRAM" error and it quits before displaying the interface.

There is a bug in this version of the drivers: they report an incorrect amount of free video memory. The workaround is to force the game to use the Apple Software engine in the interface: launch WaterRace on an ATI card, go into the Preferences dialog, turn on the "Use Apple Software engine in the interface" option, save and quit the game. Then relaunch WaterRace on the 3Dfx card.
