What is SpriteDraw:

SpriteDraw is a powerfull 2D engine based on FasterDraw. It was especially designed for PowerMacs and provide features not available in Quickdraw like aliased sprites.


Main caracteristics:

  • 16 Bits depth only.
  • can plot a sprite at any position of a pixmap.
  • 3 sprite copying modes (note: you may use different modes at the same time):
    1. Plain: copy the sprite to the destination pixmap overwriting the destination pixmap.
    2. Mask: a color is defined as transparent and a 1-Bit mask is attached to each sprite. When copying the sprite to the destination pixmap, only masked portions of the image are copied.
    3. Alpha: an 8-Bits alpha layer is attached to each sprite. When copying the sprite to the destination pixmap, the sprite image is merged with its content according to the alpha layer. This results in aliasing.
  • Fully optimized for PowerPC: SpriteDraw takes advantage of all the accelerations methods available on this plateforme: cache, pipelining, PCI bus...
  • Especially made for games.

Performances:

I hadn't time to benchmark SpriteDraw (maybe in a further update). All I can tell you is that it's pretty fast!



System requirements:
  • PowerPC processor MacOS system.
  • System 7.1 or later.

    FasterDraw FAQ:
    • Is there any limitations with SpriteDraw?
      Sprite runs only with 16 Bits graphics. But its architecture was designed to work with 8/16/24/32 bitdepth, so you shouldn't have to much work adding support for other depths.
    • Can I see an example of use of SpriteDraw?
      There is a demo application in the dev. kit, so that you can get immediately an idea of the quality and speed of SpriteDraw.
